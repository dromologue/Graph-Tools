Overview

The Graph Tools MCP Server is a Model Context Protocol (MCP) server that provides AI assistants with powerful graph analysis capabilities. It can analyze relationships in data, create interactive visualizations, and perform advanced network analysis including centrality measures.

Installation & Setup

Prerequisites

  • Node.js (v16 or higher)
  • Ruby (for graph processing)
  • Claude Desktop or compatible MCP client

Installation Steps

  1. Clone the Graph Tools repository
  2. Navigate to the mcp-graph-server directory
  3. Run npm install to install dependencies
  4. Configure your MCP client to use the server

MCP Client Configuration

Add this configuration to your Claude Desktop config file:

{ "mcpServers": { "graph-server": { "command": "node", "args": ["path/to/Graph-Tools/mcp-graph-server/index.js"], "env": {} } } }

Available Tools

analyze_relationships
Analyze relationships in data and create a comprehensive graph visualization with automatic relationship detection.

Parameters:

data array required
Array of data objects to analyze for relationships
relationship_fields array required
Fields to use for determining relationships (e.g., ["id", "parent_id"])
node_label_field string
Field to use as node labels (default: "id")
create_adjacency_matrix
Create an adjacency matrix from predefined relationship data and generate an interactive visualization.

Parameters:

relationships array required
Array of relationship objects with from/to properties
vertices array required
Array of vertex names
calculate_centrality
Calculate various centrality measures for graph nodes to identify the most important vertices in the network.

Parameters:

relationships array required
Array of relationship objects with from/to properties
vertices array required
Array of vertex names
measures array
Centrality measures to calculate: degree, betweenness, closeness, eigenvector, all (default: ["all"])
top_n number
Number of top nodes to return for each measure (default: 10)
analyze_network_structure
Comprehensive network analysis combining relationship analysis with centrality measures for complete network insights.

Parameters:

data array required
Array of data objects to analyze for relationships
relationship_fields array required
Fields to use for determining relationships
node_label_field string
Field to use as node labels (default: "id")
include_centrality boolean
Whether to include centrality analysis (default: true)

Usage Examples

Example 1: Social Network Analysis

// Example data structure const socialData = [ { id: "Alice", friends: ["Bob", "Carol"] }, { id: "Bob", friends: ["Alice", "David"] }, { id: "Carol", friends: ["Alice", "David"] }, { id: "David", friends: ["Bob", "Carol"] } ]; // Use with analyze_relationships tool { "data": socialData, "relationship_fields": ["friends"], "node_label_field": "id" }

Example 2: Organizational Hierarchy

// Example organizational data const orgData = [ { name: "CEO", reports_to: null }, { name: "CTO", reports_to: "CEO" }, { name: "Engineer1", reports_to: "CTO" }, { name: "Engineer2", reports_to: "CTO" } ]; // Use with analyze_relationships tool { "data": orgData, "relationship_fields": ["reports_to"], "node_label_field": "name" }

// Direct relationship specification { "relationships": [ { "from": "A", "to": "B", "weight": 1 }, { "from": "B", "to": "C", "weight": 2 }, { "from": "C", "to": "A", "weight": 1 } ], "vertices": ["A", "B", "C"] }

Centrality Measures Explained

Degree Centrality

Measures the number of direct connections a node has. High degree centrality indicates well-connected nodes.

Betweenness Centrality

Measures how often a node appears on shortest paths between other nodes. High betweenness indicates bridge nodes.

Closeness Centrality

Measures the average shortest path length from a node to all other nodes. High closeness indicates central positions.

Eigenvector Centrality

Measures influence based on connections to other influential nodes. High eigenvector centrality indicates prestigious connections.

Error Handling & Troubleshooting

Common Issues

  • Ruby not found: Ensure Ruby is installed and accessible in your PATH
  • Permission errors: Check that the server has write permissions to the data directory
  • Visualization not opening: Browser auto-open may fail on some systems; manually open the generated HTML file
  • Invalid data format: Ensure your data objects contain the specified relationship fields

Data Validation

The server automatically validates input data and provides detailed error messages for:

  • Missing required fields
  • Invalid data types
  • Empty or malformed relationships
  • Circular references in data
